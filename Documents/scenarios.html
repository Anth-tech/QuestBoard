Scenarios:

  1. Name: Creating a New Team
    Description:

    The process starts when the user presses the "Create Team" button. They will be prompted to fill fields relating to the team's name and purpose. Once the user submits this form, the application will ensure all necessary fields are filled and create new instance of a team within the database; if the form is not filled, it will instead display an error message stating what is missing. After team creation, the user is assigned the role of project manager and is presented with the option to invite new team members and create tasks.


  2. Name: Creating and Assigning Tasks
    Description:

    Any user who is designated as the project manager of the relevant project, or another role granted sufficient permissions, is able to see the "Create Task" button. Upon button press a form is presented with the fields necessary for task creation: task name, description, priority level, completion state, and initial team members. The task name is the only field that is an absolute requirment. When the user submits the form, if the task name field is not blank. a new task is inserted into their database and will now appear in the dashboard of their project. After the task has been created, user's with valid authority will be able to view and click an options button within the task. They will be able to assign the task to more users via a text field within this menu.


  3. Name: Updating Task Progress
    Description:

    Any user with relevant authority will be able to see the "options" button for a task from the dashboard. Clicking on this button will open a menu containing all the information pertaining to the task that the user is able to manipulate. The project manager and users granted the authority to do so can edit all facets of a task, while any user who is assigned to the task will be able to atleast view edit the task progress unless this authority has been stripped via a special role. By changing which radio button is selected and submitting the form, the user will be able to update the task's progress. This has the potential to fail if the user has the ability to edit multiple fields and provides an invalid entry to one of them.
