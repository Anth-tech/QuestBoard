Software & Hardware Requirements:

  1. Name: Cross-Browser Compatibility
    Type: Constraint
    Summary: The application must function correctly in any modern web browser.

  2. Name: Local Server Hosting with VPN Access
    Type: Constraint
    Summary: The system will be hosted on a local server and accessed remotely through a VPN connection.

  3. Name: Windows Target Platform
    Type: Constraint
    Summary: The primary target environment is Windows using a standard mouse and keyboard.

  4. Name: English Interface
    Type: Constraint
    Summary: The user interface will be provided in English.

  5. Name: Fixed Technology Stack
    Type: Constraint
    Summary: The system must be built using Java, JavaScript/TypeScript, React, Playwright, and Docker.

  6. Name: User Manual Documentation
    Type: Functional Requirement
    Summary: A short user manual will be provided to help new users get started with the system.

  7. Name: VPN Connectivity Risk
    Type: Risk
    Summary: VPN connectivity issues may prevent or disrupt remote user access, requiring setup documentation or fallback solutions.

  8. Name: Disabled Browser Notifications Risk
    Type: Risk
    Summary: Users who disable browser notifications may miss deadline reminders.

  9. Name: Hardware Failure Data Loss Risk
    Type: Risk
    Summary: Because the system is hosted locally, hardware failure could result in data loss if backups are not maintained.

  10. Name: Server Scalability Risk
    Type: Risk
    Summary: Unexpectedly large teams may strain the local server and negatively impact system performance.

  11. Name: Secure Authentication and Session Management
    Type: Functional Requirement
    Summary: The system must securely manage login and session handling to prevent unauthorized access and improper permission escalation.

    12. Communication Requirements:

  12. Name: Common Discussion Boards
    Type: Functional Requirement
    Summary: Common Discussion Boards shall allow all Team members to post and comment on general project topics.

  13. Name: Private Discussion Boards
    Type: Functional Requirement
    Summary: Private Discussion Boards shall restrict viewing and/or posting to specific roles or explicitly chosen users.

  14. Name: Task Comments
    Type: Functional Requirement
    Summary: Each task shall contain its own comment section for team feedback and updates.

    15. Completion Tracking Requirements:

  15. Name: Task Weight
    Type: Functional Requirement
    Summary: Each task shall have a configurable weight used in weighted progress bar calculations.

  16. Name: Progress Bar
    Type: Functional Requirement
    Summary: The dashboard shall display a weighted progress bar showing overall percentage of total completion of Tasks.

  17. Name: Progress Calendar
    Type: Functional Requirement
    Summary: A calendar shall display all past and upcoming Task deadlines.

  18. Name: Changeable Calendar View
    Type: Functional Requirement
    Summary: The calendar should be changeable to different views for a day, a week, a month, and a year.

  19. Name: Deadline Notifications
    Type: Functional Requirement
    Summary: The system shall send deadline reminder notifications to members assigned to incomplete Tasks.

  20. Name: Dashboard Status
    Type: Functional Requirement
    Summary: The dashboard shall display current Task statuses and deadline alerts at glance.

    21. Task Management Requirements:

  21. Name: Account Registration
    Type: Functional Requirement
    Summary: Users can register an account with an email, password, and display name.

  22. Name: Account login
    Type: Funtional Requirement
    Summary: The user can log in and out of their account.

  23. Name: Role Assignment
    Type: Functional Requirement
    Summary: Team members can be assigned roles by project managers.

  24. Name: Revoke Role
    Type: Functional Requirment
    Summary: Project managers may revoke roles.

  25. Name: Role Access
    Type: Functional Requirement
    Summary: A role designates what a team member is able to view or modify.

  26. Name: Team Creation
    Type: Functional Requirement
    Summary: Users are able to create teams and are automatically assigned as its project manager.

  27. Name: Team Invitation
    Type: Functional Requirement
    Summary: Project managers can invite users to join a team they manage.

  28. Name: Revoke Team Access
    Type: Functional Requirement
    Summary: Project managers can remove team members from their team.

  29. Name: Task Creation
    Type: Functional Requirment
    Summary: Project managers can create a task with a title, description, attached files, priority level, and deadline.

  30. Name: Task Allocation
    Type: Functional Requirment
    Summary: Project Managers can assign tasks to team members and team members may assign themselves to open tasks.

  31. Name: Revoke Task Access
    Type: Functional Requirment
    Summary: Project managers can revoke access to a task.

  32. Name: Edit Task
    Type: Functional Requirment
    Summary: Project managers edit the details within a task.

  33. Name: Task Deletion
    Type: Functional Requirment
    Summary: Project managers can delete tasks.

  34. Name: Task Progress Reporting
    Type: Functional Requirment
    Summary: Team members can mark a task's progress towards completion as not started, In Progress, or Complete.

  35. Name: Task Priority Display
    Type: Functional Requirment
    Summary: A tasks priority shall be visible on the dashboard.

  36. Name: Filtered Task Search
    Type: Functional Requirment
    Summary: Team members shall be able to search tasks by keyword, priority, assignment, and status.

  37. Name: Task Organization
    Type: Functional Requirment
    Summary: Tasks can be sorted into folders that may be restricted to roles designated by a project manager.

  38. Name: Revoke Folder Access
    Type: Functional Requirment
    Summary: Project managers may revoke an individual team members access to folder regardless of their role.

  39. Name: Revoke Discussion Board Access
    Type: Functional Requirment
    Summary: Project managers may remove individuals from a discussion board.
